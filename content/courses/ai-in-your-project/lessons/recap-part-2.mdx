## How the pieces fit

**Streaming (L04)** is the first time the model feels real-time instead of a 3-second silence. Tokens arrive over SSE, your React UI appends them, the user sees it think. Mechanically simple; experientially decisive.

**System prompts (L05)** is where you stop talking *to* the model and start talking *about* it — its role, its tone, the things it must never do. The system prompt is invisible to the user but stamped on every turn.

**Few-shot examples (L06)** is the lesson that breaks most beginners' assumptions. You think you'll explain in English; the model thinks much better when you show two or three examples of the right output. Examples > instructions, almost always.

**Feeding data (L07)** is where the model leaves vague-paragraph mode. You pull a *small, relevant slice* of your real DB into the prompt — never the whole table. The SELECT-don't-paste rule is what keeps token bills sane and answers grounded.

**Structured output (L08)** is the lesson that makes everything downstream possible. `withStructuredOutput(zSchema)` returns a typed object, not prose. From here on, the model emits values your TypeScript can trust — which is the precondition for tools, for graphs, for anything that needs to *act* on a model's answer.

<Recap>
- **Stream by default.** The user-experience delta is enormous and the API is one method swap.
- **System prompts are stamped on every turn.** Use them for role, tone, and *hard* limits. Don't repeat instructions that belong in tool descriptions or schemas.
- **Show, don't tell.** Two or three concrete examples beat a paragraph of explanation. Few-shot is cheap and disproportionately effective.
- **Pull a slice, not the table.** Feed only what's relevant to *this* turn. Schema in the system prompt, samples in user messages, never both at full size.
- **`withStructuredOutput` ends prose-parsing.** A Zod schema produces typed JSON the rest of your code can trust. From here on, English is for users and JSON is for code.
- **The five techniques compose.** A real call at the end of Part II uses all five at once: stream a structured-output response, anchored to a SELECTed slice, with a system prompt that defines the role, and one example for the harder fields.
</Recap>

## Các mảnh ghép vào nhau ra sao

**Streaming (L04)** là lần đầu tiên model có cảm giác thời gian thực thay vì 3 giây im lặng. Token đến qua SSE, UI React của bạn nối thêm, người dùng nhìn thấy nó "đang nghĩ". Về kỹ thuật thì đơn giản; về trải nghiệm thì quyết định mọi thứ.

**System prompt (L05)** là chỗ bạn ngừng nói *với* model và bắt đầu nói *về* model — vai trò của nó, giọng điệu của nó, những điều nó tuyệt đối không được làm. System prompt người dùng không thấy, nhưng được "đóng dấu" lên mọi lượt.

**Few-shot (L06)** là bài làm sụp đổ giả định của hầu hết người mới. Bạn nghĩ sẽ giải thích bằng tiếng Anh; nhưng model "nghĩ" tốt hơn nhiều khi bạn đưa hai-ba ví dụ về output đúng. Ví dụ thắng giải thích, gần như luôn vậy.

**Cho ăn dữ liệu (L07)** là chỗ model thoát khỏi chế độ "nói chung chung". Bạn kéo một *lát cắt nhỏ, đúng chủ đề* từ DB thật vào prompt — không bao giờ là cả bảng. Quy tắc *SELECT chứ đừng paste* là thứ giữ cho hoá đơn token tỉnh táo và câu trả lời được neo vào thực tế.

**Structured output (L08)** là bài mở đường cho mọi thứ phía sau. `withStructuredOutput(zSchema)` trả về một object có kiểu, không phải văn xuôi. Từ đây trở đi, model phát ra giá trị mà TypeScript của bạn có thể tin được — đó là điều kiện tiên quyết cho tools, cho graphs, cho bất cứ thứ gì cần *hành động* dựa trên câu trả lời của model.

<Recap>
- **Mặc định nên stream.** Khoảng cách trải nghiệm là khổng lồ, mà API chỉ đổi một method.
- **System prompt được đóng dấu lên mọi lượt.** Dùng nó cho vai trò, giọng điệu, và các giới hạn *cứng*. Đừng lặp lại những thứ thuộc về tool description hay schema.
- **Cho xem, đừng giảng.** Hai-ba ví dụ cụ thể đánh bại cả đoạn văn giải thích. Few-shot rẻ và hiệu quả không cân xứng.
- **Kéo lát cắt, không kéo cả bảng.** Chỉ nạp những gì liên quan đến *lượt này*. Schema trong system prompt, sample trong user message, đừng để cả hai cùng lớn.
- **`withStructuredOutput` chấm dứt việc parse văn xuôi.** Một Zod schema sinh ra JSON có kiểu mà phần còn lại của code có thể tin được. Từ đây trở đi: tiếng Anh dành cho người dùng, JSON dành cho code.
- **Năm kỹ thuật ghép được vào nhau.** Một call thực tế ở cuối Phần II dùng cả năm cùng lúc: stream một response structured-output, được neo vào lát cắt SELECT, với system prompt định nghĩa vai trò, và một ví dụ cho các trường khó.
</Recap>
