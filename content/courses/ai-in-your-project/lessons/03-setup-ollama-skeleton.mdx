Time to install something. By the end of this lesson you'll have a local model server running, a Next.js project that hits it, and a streaming "hello" coming back over an SSE response. That's the smallest possible vertical slice — every later lesson is just adding to it.

## 1. Install Ollama

Ollama is a tiny local server that loads open-weight models and exposes an HTTP API on `localhost:11434`. Install it once:

```bash
# macOS
brew install ollama                              # → installs the `ollama` CLI to /opt/homebrew/bin/ollama

# Linux
curl -fsSL https://ollama.com/install.sh | sh    # → fetches and runs the installer; binary at /usr/local/bin/ollama

# Windows: download the installer from https://ollama.com/download
```

Then start the server in a terminal you'll leave open:

```bash
ollama serve                                     # → starts HTTP server on 127.0.0.1:11434 (leave this terminal open)
```

You should see logs like `Listening on 127.0.0.1:11434`.

## 2. Pull a model

Open another terminal and pull a model. Two reasonable starting points:

- **`qwen2.5:3b`** — ~2 GB, runs on most laptops, supports tool calling. Use this if you're not sure.
- **`llama3.1:8b`** — ~5 GB, smarter, also supports tool calling. Pick this if you have at least 16 GB RAM.

```bash
ollama pull qwen2.5:3b                           # → downloads ~2 GB of weights; cached at ~/.ollama/models
```

Most open-weight models *don't* support tool calling reliably. The two above are picked because they do — which matters from lesson 9 onwards. Don't substitute without testing.

## 4. Your first handler

Create `app/api/chat/route.ts`:

```ts
import { ChatOllama } from '@langchain/ollama';  // → ChatOllama class imported

export const runtime = 'nodejs';                 // → opts this Route Handler into the Node.js runtime (not Edge)

export async function POST(req: Request) {
  const { message } = await req.json();          // → string from the client request body

  const model = new ChatOllama({                 // → ChatOllama instance pointed at local Ollama
    model: 'qwen2.5:3b',                         // → use the 3B model we just pulled
    baseUrl: 'http://localhost:11434',           // → default Ollama endpoint
    temperature: 0,                              // → deterministic output
  });

  const stream = await model.stream(message);    // → AsyncIterable<AIMessageChunk>

  const encoder = new TextEncoder();             // → TextEncoder for utf-8 → Uint8Array
  const body = new ReadableStream({              // → Web ReadableStream we'll hand to Response
    async start(controller) {
      for await (const chunk of stream) {        // → each chunk: AIMessageChunk { content: 'Hi' }
        controller.enqueue(encoder.encode(`data: ${JSON.stringify(chunk.content)}\n\n`)); // → wire format: 'data: "Hi"\n\n'
      }
      controller.enqueue(encoder.encode('data: [DONE]\n\n')); // → terminator chunk for the client
      controller.close();                        // → closes the stream cleanly
    },
  });

  return new Response(body, {                    // → Response { status: 200, body: ReadableStream }
    headers: {
      'Content-Type': 'text/event-stream',       // → tells the browser this is SSE
      'Cache-Control': 'no-cache',               // → don't cache the live token stream
      Connection: 'keep-alive',                  // → keep the socket open while streaming
    },
  });
}
```

That's it. Three concepts in twenty lines:

1. **`ChatOllama`** — the LangChain adapter pointed at our local server.
2. **`model.stream(message)`** — async iterator of token chunks.
3. **`Response` + `ReadableStream`** — the standard Web API for SSE; no extra library.

<Callout type="note" title="Why temperature: 0?">
Lower temperature = more deterministic outputs. For tool calling (lessons 9+) you almost always want `0` — you want the model to *do the same thing* given the same inputs. Crank it up only when you want creative variation, like email drafts.
</Callout>

## 5. Try it

In one terminal: `npm run dev`. In another:

```bash
curl -N -X POST http://localhost:3000/api/chat \              # → -N disables curl's buffering so SSE chunks print live
  -H 'Content-Type: application/json' \                       # → sets the request content type
  -d '{"message":"Say hi in one short sentence."}'            # → POST body the route reads as `message`
# → emits SSE chunks one at a time until 'data: [DONE]'
```

You should see SSE chunks drip in:

```
data: "Hi"
data: "!"
data: " How"
data: " are"
...
data: [DONE]
```

If you got that, the vertical slice is alive: browser → Next.js Route Handler → Ollama → model → tokens streaming back. From lesson 4 onwards, we wire this to a real React UI; from lesson 9, we attach tools that read and write your database.

## 4. Handler đầu tiên

Tạo `app/api/chat/route.ts`:

```ts
import { ChatOllama } from '@langchain/ollama';  // → ChatOllama class imported

export const runtime = 'nodejs';                 // → opts this Route Handler into the Node.js runtime (not Edge)

export async function POST(req: Request) {
  const { message } = await req.json();          // → string from the client request body

  const model = new ChatOllama({                 // → ChatOllama instance pointed at local Ollama
    model: 'qwen2.5:3b',                         // → use the 3B model we just pulled
    baseUrl: 'http://localhost:11434',           // → default Ollama endpoint
    temperature: 0,                              // → deterministic output
  });

  const stream = await model.stream(message);    // → AsyncIterable<AIMessageChunk>

  const encoder = new TextEncoder();             // → TextEncoder for utf-8 → Uint8Array
  const body = new ReadableStream({              // → Web ReadableStream we'll hand to Response
    async start(controller) {
      for await (const chunk of stream) {        // → each chunk: AIMessageChunk { content: 'Hi' }
        controller.enqueue(encoder.encode(`data: ${JSON.stringify(chunk.content)}\n\n`)); // → wire format: 'data: "Hi"\n\n'
      }
      controller.enqueue(encoder.encode('data: [DONE]\n\n')); // → terminator chunk for the client
      controller.close();                        // → closes the stream cleanly
    },
  });

  return new Response(body, {                    // → Response { status: 200, body: ReadableStream }
    headers: {
      'Content-Type': 'text/event-stream',       // → tells the browser this is SSE
      'Cache-Control': 'no-cache',               // → don't cache the live token stream
      Connection: 'keep-alive',                  // → keep the socket open while streaming
    },
  });
}
```

Vậy thôi. Ba khái niệm trong hai mươi dòng:

1. **`ChatOllama`** — adapter của LangChain trỏ tới server local của ta.
2. **`model.stream(message)`** — async iterator của các chunk token.
3. **`Response` + `ReadableStream`** — Web API chuẩn cho SSE; không cần thư viện thêm.

<Callout type="note" title="Vì sao temperature: 0?">
Temperature càng thấp = output càng tất định. Với tool calling (bài 9+), bạn gần như luôn muốn `0` — bạn muốn model *làm cùng một việc* với cùng input. Chỉ vặn cao khi bạn cần sự sáng tạo, ví dụ như soạn nháp email.
</Callout>
