## The Compose file

```yaml
# docker-compose.yml
services:

  # ── 1. Nginx reverse proxy ────────────────────────────────────────
  # The ONLY service publishing a host port. Everything else is internal.
  nginx:
    build: ./nginx
    image: coffee-nginx:${TAG:-dev}
    container_name: coffee-nginx
    ports:
      - "80:80"
    depends_on:
      web: { condition: service_started }
      api: { condition: service_healthy }
    networks: [frontend-net]
    restart: unless-stopped

  # ── 2. Next.js front-end ──────────────────────────────────────────
  web:
    build:
      context: ./web
      args:
        NEXT_PUBLIC_API_URL: /api                                   # → relative; nginx routes it
    image: coffee-web:${TAG:-dev}
    container_name: coffee-web
    environment:
      NODE_ENV: production
    networks: [frontend-net, backend-net]
    depends_on:
      api: { condition: service_healthy }
    restart: unless-stopped

  # ── 3. Node/Express API ───────────────────────────────────────────
  api:
    build: ./api
    image: coffee-api:${TAG:-dev}
    container_name: coffee-api
    env_file: .env
    environment:
      NODE_ENV: production
      DATABASE_URL: postgres://${POSTGRES_USER}:${POSTGRES_PASSWORD}@db:5432/${POSTGRES_DB}
      REDIS_URL:    redis://cache:6379
      JWT_SECRET:   ${JWT_SECRET}
    networks: [backend-net]
    depends_on:
      db:    { condition: service_healthy }
      cache: { condition: service_healthy }
    healthcheck:
      test: ["CMD", "wget", "-qO-", "http://localhost:4000/health"]
      interval: 5s
      timeout: 3s
      retries: 10
    restart: unless-stopped

  # ── 4. Postgres ───────────────────────────────────────────────────
  db:
    image: postgres:16-alpine
    container_name: coffee-db
    environment:
      POSTGRES_USER:     ${POSTGRES_USER}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
      POSTGRES_DB:       ${POSTGRES_DB}
    volumes:
      - db-data:/var/lib/postgresql/data
      - ./db/init.sql:/docker-entrypoint-initdb.d/init.sql:ro
    networks: [backend-net]
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER}"]
      interval: 5s
      timeout: 3s
      retries: 10
    restart: unless-stopped

  # ── 5. Redis ──────────────────────────────────────────────────────
  cache:
    image: redis:7-alpine
    container_name: coffee-cache
    command: ["redis-server", "--save", "60", "1", "--loglevel", "warning"]
    volumes:
      - cache-data:/data
    networks: [backend-net]
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 3s
      retries: 10
    restart: unless-stopped

# ── networks ────────────────────────────────────────────────────────
networks:
  frontend-net:
  backend-net:

# ── volumes ─────────────────────────────────────────────────────────
volumes:
  db-data:
  cache-data:
```

## The Nginx config

`nginx/nginx.conf`:

```nginx
events {}

http {
  upstream web_upstream { server web:3000; }
  upstream api_upstream { server api:4000; }

  server {
    listen 80;
    server_name _;

    location /api/ {
      proxy_pass         http://api_upstream/;
      proxy_set_header   Host              $host;
      proxy_set_header   X-Real-IP         $remote_addr;
      proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
      proxy_set_header   X-Forwarded-Proto $scheme;
    }

    location / {
      proxy_pass         http://web_upstream;
      proxy_set_header   Host              $host;
      proxy_set_header   X-Real-IP         $remote_addr;
      proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
      proxy_set_header   X-Forwarded-Proto $scheme;
    }
  }
}
```

`nginx/Dockerfile`:

```dockerfile
FROM nginx:1.27-alpine
COPY nginx.conf /etc/nginx/nginx.conf
EXPOSE 80
```

## Bringing it up

```bash
cp .env.example .env                                                 # → fill in real values
docker compose up -d --build                                          # → build images, start stack
docker compose ps                                                     # → 5 services, all healthy
curl http://localhost                                                 # → Next.js home page
curl http://localhost/api/health                                      # → API health endpoint
```

If something looks off:

```bash
docker compose logs -f api                                           # → tail one service
docker compose exec db psql -U app -d coffee -c '\dt'                # → list tables in Postgres
docker compose exec cache redis-cli ping                             # → ping Redis
docker compose config                                                # → see the rendered Compose with .env substituted
```

When you are done for the day:

```bash
docker compose stop                                                  # → stop containers, keep them around
docker compose start                                                 # → resume tomorrow, fast
docker compose down                                                  # → stop AND remove containers (volumes survive)
docker compose down -v                                               # → also drop named volumes (☢️ data loss)
```

## Biến nó thành của bạn

<EN>

Three exercises that turn this into a portfolio piece:

1. **Add a one-shot migrator service.** Re-use the `api` image, run `npm run migrate`, gate the API on `migrate: condition: service_completed_successfully`.
2. **Add Adminer or pgAdmin** behind a `profiles: [admin]` gate so it only starts when you `--profile admin up`.
3. **Add `compose.prod.yml`** that swaps `build:` for pulled `image:` from GHCR, marks volumes `external: true`, and bumps `restart: always`.

By the end you will have something I genuinely encourage you to drop into your CV. *"Designed and ran a five-service Compose stack with healthcheck-gated startup, isolated networks, persistent volumes, and a CI pipeline that builds and pushes to GHCR"* is a real sentence about a real thing.

<Recap>
- One Compose file, five services: nginx → web + api → db + cache. Only nginx publishes a port; backend services are unreachable from the host.
- Healthcheck-gated `depends_on` makes the boot order deterministic on cold machines.
- Volumes survive `down`; only `down -v` drops them. Mark critical volumes `external: true` to prevent accidents.
- `.env` for secrets, `.env.example` for shape, `docker compose config` for verification. Lesson 16 starts Part 5 with healthchecks, restart policies, and graceful shutdown — promoting this stack to something that survives in the wild.
</Recap>
