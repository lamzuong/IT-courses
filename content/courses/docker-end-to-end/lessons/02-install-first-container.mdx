<EN>

1. **`docker` CLI** — the command-line tool you type. `docker run`, `docker ps`, `docker build`. It does almost nothing on its own. It is a thin client that sends your commands to…
2. **The Docker daemon (`dockerd`)** — a long-running background process. *This* is the part that actually creates containers, manages images, isolates filesystems, attaches networks. When people say "Docker is running," they mean the daemon is running.
3. **A registry** — the remote library of images. Docker Hub is the default. GitHub has GHCR. Companies host their own. Every `docker pull` fetches from a registry.

When you type `docker run nginx`:

- The CLI sends a request to the daemon.
- The daemon checks: do I have the `nginx` image locally? No → pull it from the registry. Yes → skip.
- The daemon builds an isolated environment (filesystem, network, processes), starts nginx inside it, and prints the container ID.

That is every Docker command, ever. Knowing those three boxes is more than half the battle.

## Installing Docker

</EN>

<VI>

1. **CLI `docker`** — công cụ dòng lệnh bạn gõ. `docker run`, `docker ps`, `docker build`. Bản thân nó gần như không làm gì. Nó là client mỏng, gửi lệnh của bạn đến…
2. **Docker daemon (`dockerd`)** — một process nền chạy lâu dài. *Đây* mới là phần thật sự tạo container, quản lý image, cô lập filesystem, gắn network. Khi ai đó nói "Docker đang chạy", họ đang nói daemon đang chạy.
3. **Registry** — thư viện image ở xa. Mặc định là Docker Hub. GitHub có GHCR. Công ty có thể tự host. Mỗi lệnh `docker pull` đều kéo từ một registry.

Khi bạn gõ `docker run nginx`:

- CLI gửi yêu cầu cho daemon.
- Daemon kiểm tra: trong máy đã có image `nginx` chưa? Chưa → kéo từ registry. Rồi → bỏ qua bước kéo.
- Daemon dựng một môi trường cô lập (filesystem, network, process), khởi động nginx trong đó, in container ID ra.

Mọi lệnh Docker xưa nay đều như vậy. Hiểu ba cái hộp đó là hơn nửa trận đấu rồi.

## Cài Docker

</VI>

<Callout type="note" title="Pick your OS / Chọn hệ điều hành">
**Mac / Windows** — install **Docker Desktop** from [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop). It bundles the daemon + CLI + a small Linux VM (because the kernel namespaces Docker needs are Linux-only). After installing, open the app once so the daemon starts.

**Linux** — install the `docker` package from your distro's repo, or use the official convenience script:

```bash
curl -fsSL https://get.docker.com | sh                            # → installs docker engine
sudo usermod -aG docker $USER                                     # → run docker without sudo (log out + back in)
```

**Why a VM on Mac/Windows?** Docker containers use Linux kernel features (namespaces, cgroups). macOS and Windows do not have those. Docker Desktop quietly runs a tiny Linux VM that hosts the daemon and the containers. You will not see it; just know it is why Docker Desktop uses 1–2 GB of RAM at idle.

**Tại sao Mac/Windows phải có VM?** Container Docker dùng tính năng của kernel Linux (namespace, cgroup). macOS và Windows không có. Docker Desktop âm thầm chạy một VM Linux tí hon để host daemon và container. Bạn không thấy nó; chỉ cần biết đó là lý do Docker Desktop xài 1–2 GB RAM lúc rảnh.
</Callout>

<EN>

After installing, verify in your terminal:

```bash
docker --version                                                  # → e.g. Docker version 27.3.1
docker info                                                       # → daemon status + system info
```

If `docker info` says *"Cannot connect to the Docker daemon"*, the daemon is not running. On Mac/Windows: open Docker Desktop. On Linux: `sudo systemctl start docker`.

Verify the container actually existed for those few milliseconds:

```bash
docker ps                                                         # → empty (only RUNNING containers)
docker ps -a                                                      # → shows the exited hello-world container too
```

`docker ps` only lists *running* containers. Add `-a` to see the dead ones. The hello-world container did its job (print a message) and exited; it is still on disk until you remove it.

## A more interesting first run

`hello-world` exits immediately. Let's run something that stays alive: nginx.

```bash
docker run -d -p 8080:80 --name web nginx                         # → start nginx, detached, port-mapped
```

Open `http://localhost:8080` in your browser. You should see the nginx welcome page. Five flags worth knowing now:

- `-d` — **detached**. Run in the background. Without it, the container's logs would attach to your terminal and Ctrl-C would stop it.
- `-p 8080:80` — **port map**. *host port : container port*. Inside the container, nginx listens on 80. Outside, your browser hits 8080. Without this flag, the container is unreachable.
- `--name web` — give the container a memorable name. Without it, Docker invents something like `boring_einstein`.
- `nginx` — the image. Pulled from Docker Hub (the default registry) if you don't have it.
- *(implicit)* the daemon picks a unique container ID for you, prints it, and detaches.

</EN>

## Một lượt chạy thật hơn

`hello-world` thoát ngay. Giờ chạy thứ ở lại lâu: nginx.

```bash
docker run -d -p 8080:80 --name web nginx                         # → bật nginx, chạy nền, map port
```

Mở `http://localhost:8080` trong trình duyệt. Bạn sẽ thấy trang chào của nginx. Năm cờ nên thuộc lòng ngay:

- `-d` — **detached**. Chạy nền. Không có cờ này, log của container sẽ gắn vào terminal và Ctrl-C là tắt container luôn.
- `-p 8080:80` — **map port**. *port của host : port của container*. Trong container nginx nghe port 80. Bên ngoài, trình duyệt gõ 8080. Không có cờ này thì container không ai gọi tới được.
- `--name web` — đặt tên dễ nhớ. Không thì Docker tự đặt kiểu `boring_einstein`.
- `nginx` — tên image. Nếu máy chưa có, Docker kéo từ Docker Hub (registry mặc định).
- *(ngầm)* daemon tự sinh container ID độc nhất, in ra, rồi tách khỏi terminal.

## Looking inside

```bash
docker ps                                                         # → 'web' is listed, status: Up
docker logs web                                                   # → nginx access/error logs
docker exec -it web sh                                            # → open a shell INSIDE the container
# inside: ls, ps aux, exit
docker stop web                                                   # → graceful stop
docker rm web                                                     # → delete the stopped container
```

`exec -it` is the one to memorise. It is your X-ray vision: open a shell *inside* a running container and look around like it is its own little Linux box. `i` keeps stdin open, `t` allocates a TTY — same flags as `ssh` muscle memory.

## Nhìn vào bên trong

```bash
docker ps                                                         # → 'web' nằm trong list, status: Up
docker logs web                                                   # → log access/error của nginx
docker exec -it web sh                                            # → mở shell BÊN TRONG container
# bên trong: ls, ps aux, exit
docker stop web                                                   # → tắt êm
docker rm web                                                     # → xoá container đã tắt
```

`exec -it` là lệnh phải thuộc lòng. Nó là kính X-quang: mở shell *trong* container đang chạy và đi lại như trong một máy Linux nhỏ. `i` giữ stdin mở, `t` cấp TTY — cùng cờ với `ssh`, gõ riết quen tay.

**EN —** Think of the Docker daemon as the **barista**. The CLI is the **order ticket** you slide across the counter. The registry is the **pantry of recipes** in the back. ## Lỡ có lỗi…

</VI>

<EN>

- *"Cannot connect to the Docker daemon"* — daemon is not running. Open Docker Desktop (Mac/Win) or `systemctl start docker` (Linux).
- *"Port is already allocated"* — something on your machine already uses port 8080. Change the host side: `-p 8081:80`.
- *"permission denied while trying to connect to the Docker daemon socket"* — Linux user not in the `docker` group. `sudo usermod -aG docker $USER` and log out / log back in.
- *Pull is incredibly slow* — Docker Hub anonymous rate limit, or your network. `docker login` lifts the rate; mirrors fix the network.

<VI>

- *"Cannot connect to the Docker daemon"* — daemon chưa chạy. Mở Docker Desktop (Mac/Win) hoặc `systemctl start docker` (Linux).
- *"Port is already allocated"* — máy bạn đã có ai đó dùng port 8080. Đổi phía host: `-p 8081:80`.
- *"permission denied while trying to connect to the Docker daemon socket"* — user Linux chưa nằm trong group `docker`. Chạy `sudo usermod -aG docker $USER` rồi logout / login lại.
- *Kéo image rất chậm* — bị giới hạn ẩn danh của Docker Hub hoặc do mạng. `docker login` nâng giới hạn; dùng mirror để chữa mạng.
