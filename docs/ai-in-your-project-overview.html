Talking track — branch by branch Kịch bản thuyết trình — từng nhánh

    How the pieces connect Các mảnh ghép kết nối với nhau như thế nào

    • Input flow — your prompt + retrieved context + tool results all feed into the LLM as messages. The LLM decides what to say next or which tool to call. Luồng đầu vào — prompt + context lấy từ retrieval + kết quả tool đều đi vào LLM dưới dạng message. LLM quyết định nói gì hoặc gọi tool nào tiếp theo.
    • Orchestration — LangGraph wraps the LLM and tools in nodes + edges. interrupt() sits between the agent and the write tools so a human approves every state-changing action. Điều phối — LangGraph gói LLM và tool vào các node + edge. interrupt() nằm giữa agent và write tool để con người duyệt mọi hành động thay đổi state.
    • Knowledge loop — Embeddings turn FAQ into vectors → Vector DB stores them → RAG retrieves the top-k for each query → the LLM answers with that context attached. Vòng tri thức — Embedding biến FAQ thành vector → Vector DB lưu → RAG lấy top-k cho mỗi query → LLM trả lời với context đó kèm theo.
    • Hygiene wrap — Safety validates every input and output; Evaluation runs golden tests on every prompt change; Observability traces tokens + cost per call. Vệ sinh bao quanh — Safety kiểm tra mọi input/output; Evaluation chạy golden test mỗi lần đổi prompt; Observability trace token + chi phí từng call.
    • Deploy & cost — One VPS + three containers ships it; the AI Gateway routes between providers (cheap model for routing, expensive for answers) and fails over when one is down; a daily kill-switch caps the worst-case bill. Deploy & chi phí — Một VPS + ba container đủ để ship; AI Gateway route giữa các provider (model rẻ cho routing, đắt cho câu trả lời) và failover khi một bên chết; kill-switch hàng ngày giới hạn chi phí xấu nhất.
