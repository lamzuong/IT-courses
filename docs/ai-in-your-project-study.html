The center — LLM Agent Trung tâm — LLM Agent

the brain · Ollama (local) + hosted (cloud) bộ não · Ollama (local) + hosted (cloud)

Everything in this course orbits one idea: an LLM agent is a language model that can act, not just chat. It reads, decides, calls tools, asks for confirmation, and reports back. The eight branches around it are the eight things you must wire up to make that work in a real project. Mọi thứ trong khoá này xoay quanh một ý: một LLM agent là model ngôn ngữ có thể hành động, không chỉ chat. Nó đọc, quyết định, gọi tool, xin xác nhận, rồi báo lại. Tám nhánh quanh nó là tám thứ bạn phải lắp ráp để nó chạy trong dự án thật.

Two model worlds matter: Ollama (you run the model on your machine — cheap, slow, private) and hosted providers like OpenAI / Anthropic / Google (fast, expensive, off your premises). Production usually mixes both via an AI Gateway. Hai thế giới model quan trọng: Ollama (chạy model trên máy bạn — rẻ, chậm, riêng tư) và provider hosted như OpenAI / Anthropic / Google (nhanh, đắt, không ở máy bạn). Production thường trộn cả hai qua AI Gateway.

01 Prompting

shape behaviour định hình hành vi
"The system prompt is the agent's job description — write it like you're onboarding a new hire." "System prompt là bản mô tả công việc của agent — viết như bạn đang onboard nhân viên mới."

Why

The model has no idea what your app is, who the user is, or what tone fits. Prompting is how you give it that context before it sees the first user message. Bad prompting wastes tokens and produces wrong-shaped answers; good prompting is the cheapest engineering you can do. Model không biết app bạn là gì, user là ai, tone nào hợp. Prompting là cách bạn cho nó context đó trước khi nó nhìn thấy tin nhắn đầu tiên từ user. Prompt tệ tốn token và ra câu trả lời sai shape; prompt tốt là việc kỹ thuật rẻ nhất bạn có thể làm.

Key terms

  • System prompt Role, tone, hard limits. Loaded once per conversation. Vai trò, tone, giới hạn cứng. Nạp một lần mỗi conversation.
  • Few-shot 2–4 input→output examples that teach by demonstration. 2–4 ví dụ input→output, dạy bằng demo.
  • withStructuredOutput + Zod Forces the model to return JSON matching a schema. Prerequisite for tool calling. Ép model trả JSON khớp schema. Điều kiện tiên quyết để tool calling.
  • Schema + samples in context Paste your data shape and 1–2 real rows so answers stay grounded. Paste shape dữ liệu và 1–2 dòng thật để câu trả lời bám vào dữ liệu thật.

Talking script

  1. Open: prompts are the cheapest lever.Mở: prompt là đòn bẩy rẻ nhất.
  2. System prompt = role + tone + limits.System prompt = vai trò + tone + giới hạn.
  3. Few-shot beats long instructions when behaviour is hard to describe.Few-shot hơn hướng dẫn dài khi hành vi khó tả bằng lời.
  4. Structured output with Zod turns prose into typed JSON — this is what unlocks tools.Structured output với Zod biến prose thành JSON có kiểu — đây là thứ mở khoá tools.
  5. Watch tokens. Every line in the prompt is paid for on every call.Coi chừng token. Mỗi dòng trong prompt đều bị tính tiền mỗi lần gọi.
Remember: if the model keeps "almost" doing the right thing, the answer is usually a few-shot example — not a longer instruction. Nhớ: nếu model cứ "gần như" làm đúng, lời giải thường là thêm một few-shot — không phải viết hướng dẫn dài hơn.

02 LangGraph

orchestration điều phối
"A plain while-loop calling tools works for a demo. For anything real, you need a graph." "Vòng while gọi tool đơn giản chỉ đủ demo. Bất cứ thứ gì thật, bạn cần một graph."

Why

The naive "ask LLM → call tool → ask LLM again" loop breaks down once you need conditional paths, human approvals, retries, or state that survives a crash. LangGraph models the agent as a state machine: typed state flows through nodes connected by edges, and you can pause and resume any time. Vòng "hỏi LLM → gọi tool → hỏi LLM tiếp" sẽ vỡ khi bạn cần nhánh điều kiện, người duyệt, retry, hoặc state sống sót qua crash. LangGraph mô hình agent thành state machine: state có kiểu chảy qua các node nối bằng edge, và bạn có thể dừng / resume bất cứ lúc nào.

Key terms

  • StateGraph The whole machine. State has a TypeScript type; every node reads and writes it. Cả cỗ máy. State có type TypeScript; mọi node đọc và ghi nó.
  • Nodes Functions that transform state — typically agentNode, toolsNode, custom logic. Hàm biến đổi state — thường là agentNode, toolsNode, logic riêng.
  • Edges How control flows. Conditional edges branch on state — e.g. "if a write tool, go to confirmation." Cách control chảy. Conditional edge rẽ nhánh theo state — ví dụ "nếu là write tool, sang confirmation."
  • Checkpointer Persists state to Redis or Postgres so the graph can pause / resume / recover. Lưu state vào Redis hoặc Postgres để graph pause / resume / phục hồi.
  • compile().invoke / stream Two ways to run: invoke waits for final state; stream yields each step live. Hai cách chạy: invoke chờ state cuối; stream trả từng bước trực tiếp.
  • Read auto · Write asks Read tools run automatically; write tools route through a confirmation node first. Read tool chạy tự động; write tool đi qua node confirmation trước.

Talking script

  1. Open: plain tool-call loop hits a wall fast.Mở: vòng tool-call thuần gặp giới hạn nhanh.
  2. LangGraph = state · nodes · edges.LangGraph = state · node · edge.
  3. Conditional routing splits read (auto) from write (approval).Conditional routing tách read (tự động) khỏi write (cần duyệt).
  4. Checkpointer = pause + resume. This is what makes confirmation possible.Checkpointer = pause + resume. Đây là thứ cho phép confirmation hoạt động.
  5. Stream for live UI; invoke for batch.Stream cho UI live; invoke cho batch.
Remember: the checkpointer is not optional. Without it, the moment you call interrupt() you've lost the conversation. Nhớ: checkpointer không phải tuỳ chọn. Không có nó, ngay khi gọi interrupt() bạn mất conversation.

03 Tools

what the LLM can do sức mạnh thực sự
"A chatbot talks. An agent acts. Tools are the difference." "Chatbot chỉ nói. Agent hành động. Tools là khác biệt."

Why

The LLM by itself only produces text. Tools are functions you let it call — query the database, send an email, charge a card. The contract is a Zod schema: name, description, input shape. The model picks which tool to call and what arguments to pass; your code executes it. LLM một mình chỉ sinh text. Tools là các function bạn cho phép nó gọi — query database, gửi email, tính tiền thẻ. Hợp đồng là một Zod schema: tên, mô tả, shape input. Model chọn tool nào gọi và truyền tham số gì; code của bạn thực thi.

Key terms

  • tool() + Zod The wrapper. Zod schema is the contract — the model sees it and matches arguments to it. Wrapper. Zod schema là hợp đồng — model nhìn thấy và khớp tham số theo nó.
  • Read tools Safe to run automatically. Always paginate and filter — don't return 10k rows. An toàn để chạy tự động. Luôn phân trang và filter — đừng trả về 10k dòng.
  • Write tools Change state. Treat them like a "submit" button — they need confirmation. Thay đổi state. Đối xử như nút "submit" — cần xác nhận.
  • Idempotency key A unique id per logical action. Retrying with the same key must not duplicate the effect. Id duy nhất cho mỗi hành động logic. Retry cùng key không được nhân đôi tác động.
  • Narrow · idempotent · clear Design rule: one job per tool, safe to retry, name + description leaves no doubt. Quy tắc thiết kế: một việc mỗi tool, retry an toàn, tên + mô tả không gây hiểu nhầm.
  • Per-user allowlist Don't expose every tool to every user. Permission check before adding to the toolset. Đừng phơi bày mọi tool cho mọi user. Check quyền trước khi thêm vào toolset.

Talking script

  1. Open: tools turn a chatbot into an agent.Mở: tools biến chatbot thành agent.
  2. tool() + Zod schema = contract.tool() + Zod schema = hợp đồng.
  3. Read vs Write. Read = query. Write = the danger zone.Read vs Write. Read = query. Write = vùng nguy hiểm.
  4. Idempotency keys + retries so duplicates and transient failures don't break anything.Idempotency key + retry để duplicate và lỗi nhất thời không phá hỏng gì.
  5. Allowlist per user — never trust the LLM to enforce permissions.Allowlist theo user — đừng tin LLM tự kiểm tra quyền.
Remember: the tool description is part of the prompt. The model only knows what you wrote in it — so write it like you're explaining to a junior. Nhớ: mô tả tool là một phần của prompt. Model chỉ biết những gì bạn viết trong đó — viết như đang giải thích cho junior.

04 Confirmation

human-in-the-loop người trong vòng lặp
"Before any write — pause, ask, then act. This one pattern is what makes agents production-safe." "Trước mọi write — dừng, hỏi, rồi làm. Một pattern này là thứ làm cho agent dùng được trong production."

Why

LLMs hallucinate. They will, eventually, propose an action you don't want — delete the wrong row, email the wrong person. Confirmation is the safety brake: every state-changing action gets a human signature before it executes. Without it, you cannot ship to real users. LLM hallucinate. Sớm muộn nó sẽ đề xuất một hành động bạn không muốn — xoá nhầm dòng, gửi email sai người. Confirmation là phanh an toàn: mọi hành động thay đổi state đều có chữ ký người trước khi thực thi. Không có nó, bạn không thể ship cho user thật.

Key terms

  • interrupt() LangGraph call that pauses execution and saves state to the checkpointer. Hàm của LangGraph dừng thực thi và lưu state vào checkpointer.
  • Saved state Everything needed to resume — last messages, pending tool call, user context. Mọi thứ cần để resume — message cuối, tool call đang chờ, context user.
  • Approve / Reject / Edit Three UI choices for the human: run as-is, cancel, or tweak the arguments first. Ba lựa chọn UI cho người: chạy nguyên, huỷ, hoặc sửa tham số trước.
  • Resume Load saved state, splice in the decision, continue from the node after interrupt(). Nạp state đã lưu, chèn quyết định, chạy tiếp từ node sau interrupt().
  • Audit log Every approval (who, when, what was proposed, what was approved) — non-negotiable for compliance. Mỗi lần duyệt (ai, khi nào, đề xuất gì, duyệt gì) — không thể bỏ với compliance.

Talking script

  1. Open: hallucination is not "if" but "when".Mở: hallucination không phải "nếu" mà là "khi nào".
  2. interrupt() before any write tool — pause and surface the action.interrupt() trước mọi write tool — dừng và hiện hành động ra.
  3. Human sees approve / reject / edit.Người dùng thấy duyệt / từ chối / sửa.
  4. Resume with the decision baked in.Resume với quyết định đã bake vào.
  5. Log everything. One row per approval, forever.Log mọi thứ. Một dòng mỗi lần duyệt, mãi mãi.
Remember: if a user can "approve all" without reading, you've built the wrong UI. Make the proposed action impossible to miss. Nhớ: nếu user có thể "duyệt tất cả" mà không đọc, bạn xây UI sai. Làm cho hành động đề xuất không thể bỏ qua.

05 Embeddings

meaning as vectors ý nghĩa thành vector
"Embeddings turn 'similar meaning' into 'nearby points'. That single shift unlocks search, RAG, clustering." "Embeddings biến 'ý nghĩa giống nhau' thành 'điểm gần nhau'. Một thay đổi nhỏ đó mở khoá search, RAG, clustering."

Why

Keyword search misses synonyms ("buy" vs "purchase"), languages, paraphrases. Embeddings give every piece of text a coordinate in a high-dimensional space — and two texts about the same idea land near each other regardless of word choice. That's the prerequisite for everything in retrieval. Keyword search bỏ sót đồng nghĩa ("mua" vs "tậu"), ngôn ngữ khác, paraphrase. Embeddings cho mỗi đoạn text một toạ độ trong không gian nhiều chiều — và hai text cùng ý nghĩa rơi gần nhau bất kể chọn từ. Đó là điều kiện tiên quyết cho mọi thứ trong retrieval.

Key terms

  • Vector An array of floats. Length is fixed by the model (e.g. 1024). Mảng float. Độ dài cố định theo model (ví dụ 1024).
  • Dim (dimension) How many numbers in the array. 1024 / 1536 / 3072 are common. Higher = more nuance + more storage. Bao nhiêu số trong mảng. 1024 / 1536 / 3072 hay gặp. Cao hơn = tinh tế hơn + tốn storage hơn.
  • Cosine vs dot vs L2 Three ways to measure "closeness". Pick cosine for text and never change. Ba cách đo "gần". Chọn cosine cho text và đừng đổi.
  • Normalize Scale every vector to length 1. After this, cosine and dot give the same answer. Co mọi vector về độ dài 1. Sau bước này, cosine và dot cho cùng kết quả.
  • Model choice text-embedding-3-* for English; bge-m3 for Vietnamese / multilingual. text-embedding-3-* cho tiếng Anh; bge-m3 cho tiếng Việt / đa ngữ.
  • Matryoshka truncation Some models let you keep just the first N dims and still work — like a Russian doll. Saves storage. Vài model cho phép giữ N chiều đầu mà vẫn chạy — như búp bê Nga. Tiết kiệm storage.

Talking script

  1. Open: meaning becomes geometry.Mở: ý nghĩa trở thành hình học.
  2. Vector + dim — fixed-length array of floats.Vector + dim — mảng float độ dài cố định.
  3. Cosine is the metric. Always.Cosine là metric. Luôn luôn.
  4. Normalize once at write-time. Then your search becomes a cheap dot product.Normalize một lần khi ghi. Rồi search trở thành dot product rẻ tiền.
  5. Pin your model. Different model = different space — old and new vectors don't talk to each other.Pin model. Model khác = không gian khác — vector cũ và mới không nói chuyện được với nhau.
Remember: the day you change the embedding model is the day you re-index everything. Pin it on day one. Nhớ: ngày bạn đổi embedding model là ngày bạn phải index lại toàn bộ. Pin nó từ ngày đầu.

06 RAG / Retrieval

feed it knowledge cấp tri thức
"The model doesn't know your data. RAG is how you put the right paragraph in front of it before it answers." "Model không biết dữ liệu của bạn. RAG là cách bạn đặt đoạn văn đúng trước mặt nó trước khi nó trả lời."

Why

Stuffing your whole document corpus into every prompt is impossible (context limit) and expensive (tokens). RAG = Retrieval-Augmented Generation: at query time, find the most relevant chunks of your data, paste just those into the prompt, then let the model answer with that context. Cheap, scalable, accurate. Nhét toàn bộ tài liệu vào mỗi prompt là không thể (giới hạn context) và đắt (token). RAG = Retrieval-Augmented Generation: lúc query, tìm các chunk liên quan nhất, paste chỉ chừng đó vào prompt, rồi cho model trả lời với context đó. Rẻ, scale được, chính xác.

Key terms

  • Chunking Split docs into pieces. Fixed-size (simple), recursive (respects structure), semantic (slowest, best quality). Cắt tài liệu. Cố định (đơn giản), recursive (theo cấu trúc), semantic (chậm nhất, chất lượng nhất).
  • Index type flat (exact, slow), IVF (cluster-based), HNSW (graph, fast — the default). flat (chính xác, chậm), IVF (theo cụm), HNSW (graph, nhanh — default).
  • pgvector vs Qdrant pgvector = your existing Postgres. Qdrant = dedicated vector DB, faster at scale. Start with pgvector. pgvector = chính Postgres bạn đang có. Qdrant = DB vector chuyên dụng, scale nhanh hơn. Bắt đầu bằng pgvector.
  • Hybrid retrieval Vector (meaning) + keyword (exact match like SKUs) merged via Reciprocal Rank Fusion. Vector (ý nghĩa) + keyword (khớp chính xác như SKU) trộn qua Reciprocal Rank Fusion.
  • Recall@k · MRR Recall@k = "is the right chunk in the top k?". MRR = how high up it sits on average. Recall@k = "chunk đúng có trong top k không?". MRR = trung bình nó nằm ở vị trí thứ mấy.
  • Reranker A second, slower model that re-orders the top 20 → top 5 by careful relevance scoring. Model thứ hai chậm hơn, sắp lại top 20 → top 5 bằng cách chấm điểm relevance kỹ hơn.
  • Idempotent ingestion Re-running the indexer must not duplicate chunks. Hash content + upsert by hash. Chạy lại indexer không được nhân đôi chunk. Hash content + upsert theo hash.

Talking script

  1. Open: the model doesn't know your data.Mở: model không biết dữ liệu của bạn.
  2. Chunk → embed → index → retrieve. Four steps, always in that order.Chunk → embed → index → retrieve. Bốn bước, luôn theo thứ tự đó.
  3. Hybrid retrieval because exact-match terms (codes, names) need keyword search too.Hybrid retrieval vì các từ phải khớp chính xác (mã, tên) cần keyword search.
  4. Measure recall@k. If retrieval is wrong, no prompt can save you.Đo recall@k. Nếu retrieval sai, không prompt nào cứu nổi.
  5. Rerank the top 20 → top 5 only if retrieval alone isn't precise enough.Rerank top 20 → top 5 chỉ khi retrieval một mình chưa đủ chính xác.
Remember: a wrong answer is almost always a retrieval bug, not a prompt bug. Look at what got retrieved before you touch the prompt. Nhớ: câu trả lời sai gần như luôn là bug retrieval, không phải bug prompt. Xem cái gì được lấy ra trước khi sửa prompt.

07 Safety & Eval

guards · golden tests rào chắn · golden test
"LLMs are non-deterministic. The fix isn't 'better prompts' — it's guards on input, validation on output, tests on both." "LLM không tất định. Cách chữa không phải 'prompt tốt hơn' — mà là rào input, validate output, test cả hai."

Why

The same prompt can produce different outputs on different days. Users can paste malicious instructions into a chat box ("ignore everything above and…"). Without safety and eval, your agent looks fine in dev and silently regresses in prod. This branch is the engineering discipline layer. Cùng một prompt có thể ra output khác nhau ở các ngày khác nhau. User có thể paste chỉ thị độc hại vào chat ("bỏ qua mọi thứ ở trên và…"). Không có safety và eval, agent của bạn nhìn ổn ở dev và lặng lẽ regress ở prod. Nhánh này là lớp kỷ luật kỹ thuật.

Key terms

  • Prompt injection guard Strip / detect attempts to override the system prompt from user input. Loại / phát hiện nỗ lực ghi đè system prompt từ input user.
  • Output validation Run every LLM response through schema.parse() — reject hallucinated shapes immediately. Chạy mỗi response qua schema.parse() — từ chối shape ảo ngay lập tức.
  • PII redaction Strip emails, phone numbers, names before writing to logs or sending to external providers. Bỏ email, số điện thoại, tên trước khi ghi log hoặc gửi đến provider ngoài.
  • Golden test A locked input → expected output pair. Re-run on every prompt change to catch regressions. Cặp input khoá → output mong đợi. Chạy lại mỗi khi đổi prompt để bắt regression.
  • Snapshot regression Diff today's output against yesterday's "good" snapshot. Loud alarm when they differ. Diff output hôm nay với snapshot "tốt" hôm qua. Báo động to khi khác nhau.
  • Fake LLM A test double that returns canned responses. Lets you run thousands of tests without API cost. Double test trả response có sẵn. Cho phép chạy hàng nghìn test không tốn API.

Talking script

  1. Open: LLMs are non-deterministic — you have to engineer around that.Mở: LLM không tất định — phải kỹ thuật quanh nó.
  2. Input guards — block injection and obviously malicious patterns.Input guard — chặn injection và pattern độc rõ ràng.
  3. Output validation with Zod — every response.Output validation với Zod — mọi response.
  4. Redact PII before logging or before any third-party hop.Redact PII trước khi log hoặc đi qua bên thứ ba.
  5. Golden tests + fake LLMs — cheap CI that catches prompt drift.Golden test + fake LLM — CI rẻ bắt được prompt drift.
Remember: if your CI doesn't have at least 5 golden tests for the agent, you'll find out about prompt regressions from your users — never let that happen. Nhớ: nếu CI không có ít nhất 5 golden test cho agent, bạn sẽ biết về prompt regression qua user — đừng để chuyện đó xảy ra.

08 Deploy & Observability

ship it · watch it ship · giám sát
"You don't need Kubernetes. You need three containers, an AI Gateway, and a kill-switch." "Bạn không cần Kubernetes. Bạn cần ba container, một AI Gateway, và một kill-switch."

Why

The minimum viable production setup for an agent is much smaller than people assume. A single VPS running three containers (your app, Postgres, Qdrant) handles real traffic for weeks. The bigger risks are cost runaway and silent breakage — both solved by an AI Gateway and tracing. Setup production tối thiểu cho agent nhỏ hơn nhiều so với mọi người nghĩ. Một VPS chạy ba container (app của bạn, Postgres, Qdrant) gánh được traffic thật vài tuần. Rủi ro lớn hơn là chi phí mất kiểm soát và hỏng âm thầm — cả hai đều giải quyết bằng AI Gateway và tracing.

Key terms

  • 3-container VPS app · postgres (with pgvector) · qdrant. One docker-compose, one machine, week-one ready. app · postgres (kèm pgvector) · qdrant. Một docker-compose, một máy, sẵn sàng tuần đầu.
  • AI Gateway A proxy in front of all providers. Unifies billing, adds retries, lets you swap models without code changes. Proxy đứng trước mọi provider. Gộp hoá đơn, thêm retry, cho phép đổi model không sửa code.
  • Routing Cheap model classifies the intent; expensive model writes the answer. Saves ~70% on bills. Model rẻ phân loại ý định; model đắt viết câu trả lời. Tiết kiệm ~70% hoá đơn.
  • Failover When primary provider 500s or rate-limits, gateway transparently retries on a fallback. Khi provider chính 500 hoặc rate-limit, gateway retry trong suốt sang fallback.
  • Tracing Every call captured — tokens in, tokens out, latency, which tool fired, which user. Searchable. Mỗi call lưu lại — token vào, token ra, latency, tool nào chạy, user nào. Tìm kiếm được.
  • Kill-switch A daily budget cap. When today's spend exceeds X, the gateway returns errors until tomorrow. Giới hạn ngân sách ngày. Khi chi tiêu hôm nay vượt X, gateway trả error đến mai.
  • Canary ramp Send the new version to 5% → 50% → 100% of users with a fast rollback path. Đẩy version mới cho 5% → 50% → 100% user, có đường lùi nhanh.

Presentation flow — how to chain the eight pieces Mạch trình bày — cách nối tám phần lại

When you present the whole picture, don't go branch-by-branch — that sounds like a list. Tell it as a story in this order: Khi bạn trình bày toàn bộ, đừng đi từng nhánh — sẽ giống đọc list. Kể như một câu chuyện theo thứ tự này:

  1. Start with the agent. "An LLM agent is a model that acts, not just chats." Bắt đầu từ agent. "LLM agent là model hành động, không chỉ chat."
  2. How does it know what to do? → Prompting. System prompt + structured output. Nó biết phải làm gì bằng cách nào? → Prompting. System prompt + structured output.
  3. How does it act? → Tools. Functions wrapped in Zod schemas. Nó hành động thế nào? → Tools. Function bọc trong Zod schema.
  4. How do all those pieces flow? → LangGraph. State machine, conditional edges, checkpointer. Các phần đó nối nhau thế nào? → LangGraph. State machine, conditional edge, checkpointer.
  5. How do we keep it safe? → Confirmation. Pause before any write; human approves. Làm sao giữ an toàn? → Confirmation. Dừng trước mọi write; người duyệt.
  6. How does it know your data? → Embeddings + RAG. Vectors, hybrid retrieval, recall@k. Nó biết dữ liệu của bạn thế nào? → Embeddings + RAG. Vector, hybrid retrieval, recall@k.
  7. How do we stop it breaking? → Safety & Eval. Guards, validation, golden tests. Làm sao tránh nó hỏng? → Safety & Eval. Guard, validation, golden test.
  8. How do we ship it? → Deploy & Observability. 3 containers, AI Gateway, kill-switch, canary. Ship thế nào? → Deploy & Observability. 3 container, AI Gateway, kill-switch, canary.
  9. Close: "These eight pieces aren't optional — they're what 'an agent in production' actually means." Kết: "Tám phần này không phải tuỳ chọn — đó chính là nghĩa của 'agent trong production'."
