Why Vercel for this project

Vercel is the company behind Next.js, the framework your IT-courses site is built with. Hosting Next.js on Vercel is a bit like running an Apple app on an iPhone — the platform is designed around the framework, so there is no configuration, no PM2, no Caddy, no nginx, no SSH keys, no server to keep patched. You connect a GitHub repository, you click Deploy, and a minute later your site is live on the internet with HTTPS already turned on.

03

Push your project to GitHub

Vercel deploys from a Git repository. You need one. If you already have your project on GitHub, skip this step and head straight to Step 04.

3.1 — Create a new repo on GitHub

Go to github.com/new. Name it it-courses (or anything). Set it to Private — there's no downside; Vercel reads private repos just fine. Don't add a README, .gitignore, or license — your project already has them.

3.2 — Connect your local project

In PowerShell, from the project root:

# initialize git if not already done
git init
git add .
git commit -m "initial commit"

# point at your new GitHub repo (replace YOUR-USERNAME)
git branch -M main
git remote add origin https://github.com/YOUR-USERNAME/it-courses.git
git push -u origin main

GitHub will probably ask you to authenticate. The easiest path: install GitHub CLI and run gh auth login once; it handles the credentials for you across all future pushes.

!

Never commit secrets. Your .gitignore already excludes .env* files — keep it that way. API keys go into Vercel's environment variable UI in Step 10, never into the repo.

04

Create your Vercel account

Go to vercel.com/signup. Click Continue with GitHub. This is the easiest path because Vercel can read your repo list directly — no manual integration setup later.

  • Pick a username (lowercase, hyphens — this becomes the URL prefix for your free *.vercel.app deployments).
  • Pick your country and select Hobby at signup. You can upgrade to Pro in Step 07 once the first deploy works.
  • Email verification — check your inbox, click the link.
i

You can also sign in with GitLab, Bitbucket, SAML SSO, or email. GitHub is just the smoothest for first-timers because we're using GitHub as the source repo.

05

Import the project

From your Vercel dashboard, click Add New… → Project. You'll see a list of your GitHub repositories. Find it-courses and click Import.

5.1 — First-time GitHub authorization

If this is your first import, GitHub will pop a permission screen asking which repos Vercel can read. Pick Only select repositories and tick just it-courses. This way Vercel only sees the one repo, not your entire GitHub account.

✓

07

Upgrade to Pro ($20/mo)

Now that the site is deployed and you've seen it work, upgrade to Pro to unlock commercial use, Speed Insights, and the higher bandwidth ceiling.

7.1 — Open billing

Top-right avatar → Settings → Billing in the left sidebar. Click Upgrade to Pro.

7.2 — Pick a payment method

Enter your card details. Vercel charges $20 immediately and again on the same calendar day each month. You'll get an email receipt every cycle. Vietnamese-issued cards work — Vercel uses Stripe under the hood.

7.3 — Set a spend cap (very important)

Pro is "$20 base + metered overage" if you blow past the included quotas. The cap is the safety wire that prevents that:

Settings → Billing → Spend management → Pause projects when limit is reached. Set the cap to $20. Now even in the unlikely case of a traffic spike, Vercel pauses serving rather than charging you more.

!

The cap is not optional. Without it, a viral moment or a misconfigured loop could push your bill past your budget. With it, the worst case is "the site went down for the rest of the month" — annoying, but never expensive.

08

Connect your custom domain

You bought a domain — let's wire it up.

8.1 — Add the domain in Vercel

From your project dashboard: Settings → Domains. Type your domain (e.g. yourname.com) and click Add.

If you want both the root yourname.com and the www.yourname.com subdomain, add both. Vercel will offer to auto-redirect one to the other — pick your preferred canonical version (most people pick the bare domain).

8.2 — Vercel shows you the DNS records to add

The screen will show one of two things depending on what you typed:

If you added…Record typeValue to set at your registrar
yourname.com (root)A76.76.21.21
www.yourname.comCNAMEcname.vercel-dns.com

Copy these — you'll paste them at your registrar in the next step.

09

Add DNS records at your registrar

Where you bought the domain (Namecheap, GoDaddy, Cloudflare, Tenten.vn, Mat Bao, P.A. Việt Nam, etc.), log in and find the DNS or Manage Records page for your domain.

9.1 — Delete the placeholder records first

Most registrars start you with a default "parking page" A record. Delete it. Otherwise it will conflict with Vercel's.

9.2 — Add the records Vercel gave you

TypeName / HostValueTTL
A@ (or blank, depending on registrar)76.76.21.21Auto / 3600
CNAMEwwwcname.vercel-dns.comAuto / 3600

Save. DNS propagation usually takes 5-30 minutes; sometimes a few hours globally, but Vercel will pick it up almost immediately.

9.3 — Verify in Vercel

Back in Settings → Domains, your domain will go from a yellow ⚠ "Invalid Configuration" to a green ✓ "Valid Configuration" once it sees the record. SSL certificates are generated automatically — no manual action needed. Visit https://yourname.com and your site loads.

i

Cloudflare users: if your registrar is Cloudflare and you're proxying (orange cloud), switch the proxy to DNS only (grey cloud) for both records — Vercel handles its own CDN, double-proxying breaks things.

10

Environment variables (skip if you have none)

Your IT-courses site is mostly static MDX content — it probably doesn't need any secrets to build or run. But the AI-in-your-project course content references env vars like ANTHROPIC_API_KEY. If you ever wire up a live demo that calls an API, this is where you'd add the key.

10.1 — Where to add them

Settings → Environment Variables. For each variable:

  • Name: e.g. ANTHROPIC_API_KEY
  • Value: paste the secret
  • Environments: tick Production, Preview, and optionally Development. Use different keys per environment — never reuse a production key in preview.

10.2 — Pull them locally with Vercel CLI

If you want the same env vars on your laptop, install the CLI once: npm i -g vercel, then in the project root run vercel link (once), then vercel env pull .env.local any time you change a value. The file is in your .gitignore — it stays off GitHub.

11

How auto-deploys work

This is the part that feels like magic the first time you see it. Vercel watches your GitHub repo. Every push triggers a build.

Action in GitHubWhat Vercel doesResulting URL
Push to mainBuilds and promotes to productionyourname.com (overwritten)
Push to any other branchBuilds a Preview deploymentit-courses-git-feature-abc.vercel.app
Open a Pull RequestBuilds a Preview + posts the URL as a PR commentUnique per PR, isolated

Preview URLs are the killer feature. You can ship a draft course on a branch, share the preview link with a friend, get feedback, merge to main only when it's ready. No more "is this what the live site will look like" guesswork — the preview is the live site, on a different URL.

★

Workflow you'll fall into. Edit a lesson locally → git checkout -b draft-new-lesson → push → Vercel comments a preview URL on the PR → review on your phone → merge → production updates in 60 seconds. That's it.

12

Turn on Analytics & Speed Insights

Pro includes Vercel's privacy-friendly analytics and a Core Web Vitals dashboard. Enable them in two clicks.

12.1 — Web Analytics

Project → Analytics → Enable. Vercel automatically injects the tracking — no code change needed for most Next.js projects. You'll see page views, top pages, top referrers, top countries, all without cookies.

12.2 — Speed Insights

Project → Speed Insights → Enable. This measures real users' Core Web Vitals (LCP, INP, CLS) from their actual browsers. If a lesson is slow on mobile in Vietnam, you'll see it here within hours.

If Vercel asks you to install the package: npm i @vercel/analytics @vercel/speed-insights, then add <Analytics /> and <SpeedInsights /> to your root layout. Five lines total.

14

Rollback when something breaks

Shipped a broken commit? Don't panic. Don't git revert first. Roll back to the previous deployment in three clicks:

  • Vercel dashboard → Deployments tab
  • Find the last known-good deployment (green checkmark)
  • Click ⋯ → Promote to Production

Your domain points at the old build within seconds. The bad deployment isn't deleted — it stays in the list so you can debug it. Once you've fixed the code, push the fix and the new build automatically becomes production.

i

Why this matters. The instant-rollback button is what makes shipping feel safe. On AWS Lightsail you'd be SSHing into a server at 11pm rolling back PM2; here you're three clicks away from the previous version, no matter how badly the latest commit broke things.

16

Troubleshooting

Build fails on Vercel but works locally

Almost always one of three things:

  • Different Node version. Check Settings → Build & Development Settings → Node.js Version. Set it to match your local: run node -v on your laptop, then pick the matching version (e.g. 22.x) in Vercel.
  • Missing env var. The build references process.env.SOMETHING that you didn't add in Step 10. Add it and redeploy.
  • Case-sensitive imports. Windows is case-insensitive; Linux (Vercel's build environment) is not. import x from './Foo' works locally but fails on Vercel if the file is actually foo.tsx. Match the casing exactly.
Domain shows "Invalid Configuration" forever

DNS hasn't propagated yet, OR your records are slightly wrong. Open PowerShell and run:

nslookup yourname.com

If it returns 76.76.21.21, DNS is correct — wait 10 more minutes and refresh Vercel. If it returns something else, fix the A record at your registrar. Some registrars also need the www CNAME entered without the trailing dot.

I pushed but the site didn't update

Check the Deployments tab. If you don't see a new deployment after a push, the GitHub integration isn't wired correctly. Go to Settings → Git, click Disconnect, then re-connect the repository. Future pushes will trigger builds again.

The site loads but a specific page 404s

For Next.js App Router (which IT-courses uses), every URL must correspond to a folder under app/ or a route under content/. If a lesson was added but the dynamic route isn't picking it up, check that the MDX file is being read at build time — look at the build logs in Vercel for "Generated X static pages" and confirm your lesson is in that count.

Will my Vietnamese-issued bank card work?

Yes. Vercel uses Stripe, which accepts cards from every major Vietnamese bank (Vietcombank, Techcombank, MB Bank, ACB, VPBank, etc.) as long as they're enabled for international online payments. If a charge fails, your bank usually sends an OTP to confirm — approve it and retry. Vercel also accepts PayPal in some regions.

How do I add a teammate later?

Settings → Members → Invite. Type their email, pick their role (Member is usually right), they get an invite, they sign in with GitHub, done. Pro includes up to 10 members at no extra cost.

Can I keep both AWS Lightsail and Vercel guides for learning?

Absolutely — they teach different things. AWS teaches you Linux, PM2, Caddy, SSH, system administration; Vercel teaches you the modern "platform-as-a-service" model. For a beginner shipping a Next.js site, Vercel is the obvious answer. For understanding how websites actually work under the hood, going through the Lightsail guide once is invaluable.

How do I cancel and stop being charged?

Settings → Billing → Downgrade to Hobby. Your $20/mo stops at the end of the current cycle. Your site keeps running on Hobby limits — bandwidth drops to 100 GB but the URL and domain stay live. To delete the project entirely: Project Settings → General → Delete Project.

17

What to learn next

You now have a production Next.js site behind a real domain, with HTTPS, a global CDN, automatic deploys, preview URLs, and analytics — at a flat $20/month. From here, three good directions to grow into:

Vercel-specific

  • Branch protection. Configure GitHub so PRs must be reviewed before merging to main — prevents accidental production pushes.
  • Vercel CLI. Install npm i -g vercel for vercel logs, vercel env pull, and vercel deploy from your terminal.
  • Web Analytics dashboards. Set up an alert when bandwidth use crosses 70% of the included quota.
  • Vercel Marketplace. If you ever need a database (e.g. Neon Postgres for user accounts), add it through the marketplace and Vercel auto-wires the env vars.

Performance

  • Image optimization. Next.js's <Image> component is already optimized on Vercel — use it everywhere instead of raw <img>.
  • Static MDX. Your lessons are static — make sure they're being statically generated at build time (you'll see them in the build log under "○ Static" rather than "ƒ Dynamic").

Beyond a course site

  • User accounts. When you want sign-ups, add Clerk or Auth0 through Vercel Marketplace — five minutes from "I want users to log in" to "users are logging in."
  • Course progress tracking. A Neon Postgres database + a single API route in your Next.js app, and you can save which lessons each user has completed.
  • AI features. Lesson 24 of your AI course teaches the deployment side. With this guide done, you're ready to actually ship one.
